当前能力
自然语言查询 直接用中文提问，系统会自动理解你要查询的本体、版本或治理主题，并返回可追溯的结果。
可回答范围 支持社区高星版本、官方推荐版本、版本时间线、版本内容、版本差异和治理缺口等查询。
安全边界 查询台只读取版本库信息，不会修改版本、提交推荐或执行治理动作。
查询能力
get_community_top_version
查询某个本体文件当前社区星标最高的版本。
{"project_id":"demo","filename":"student.json"}
get_official_recommendation
查询某个本体文件当前官方推荐的版本。
{"project_id":"demo","filename":"student.json"}
get_file_timeline
查询某个本体的版本时间线、最新版本和最近变化。
{"project_id":"demo","ontology_name":"学校","limit":10}
get_version_content
读取某个本体当前工作区内容，或读取指定历史版本内容。
{"project_id":"demo","ontology_name":"学校","version_id":2}
compare_versions
比较某个本体两个版本之间的 JSON 字段差异。
{"project_id":"demo","ontology_name":"学校","left_version_id":2,"right_version_id":4}
find_governance_gaps
扫描项目或单个本体的治理缺口，例如官方推荐缺失、官方社区分歧、概率缺失或偏低。
{"project_id":"demo","ontology_name":"学校"}
使用流程
1. 提问 输入你关心的项目、本体或版本问题，也可以只写对象名，例如“学校”。
2. 识别 系统会匹配本体名称、版本号和查询意图，选择合适的数据查询路径。
3. 查询 结果来自版本库、推荐轨道和缓存索引，优先使用快速索引并保留降级查询能力。
4. 回答 页面会同时展示结论、关键证据和原始结构化结果，方便复核。
